Stuff I'm interested in

AI models in particular LLM like ChatGPT have become a large part of my every-day - whether its recipe inspiration on what to make for dinner to code assistance.

Aim

This project was created as part of nology's pre-course challenges. It was crafted using HTML, CSS, JavaScript, and the Instant-ID AI via the Replicate API. The primary goal was to develop a web app dedicated to AI while learning more about working with open source API and practicing API calls.

AI Models

As I was also keen to explore alternatives to OpenAI, it was a great chance for me to play around with replicate.com, which offers a range of exciting open-sourced models to integrate into your web projects.

Replicate

Replicate is a cloud-based platform designed to simplify the execution of machine learning models. With just a few lines of codes, users can run open-source models via API support or directly from their code, eliminating the need for extensive machine learning knowledge or additional server setup.

InstantID

InstantID is an AI model designed for user-friendly operation, requiring minimal fine-tuning. It swiftly generates images while preserving the distinctive features and characteristics of the original picture.

Where to start

This project has reinforced the importance of consulting documentation. From authentication to getting a predication, the HTTP API reference helped to work out the parameters of my request including endpoints, authorization requirements and request inits.

Challenges

1. Overcoming CORS

As I have mainly worked on fullstack projects with APIs, this was the first time I encountered a CORS policy error: No "Access-Control-Allow-Origin" header. It seems like a common problem faced with beginner engineers so there was lots of support on possible fixes. The route I decided to go down was creating my own proxy server with the help of cors-anywhere and deployed via Heroku.

2. Testing out the AI via cURL

At first, I set up my "Post" request via teh browser's Fetch API and I was having trouble with authorization. I switched to cURL and then utilized the exact example that was given in the documentation.
